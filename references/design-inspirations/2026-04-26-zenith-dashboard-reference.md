# Zenith Dashboard Reference

Date captured: 2026-04-26

## What This Is

A dashboard reference image shared during Olympus design work. The exact image file has not been archived into the repo yet, but the design reference itself is being preserved here so we can revisit it later.

## What Feels Useful

- dense operational layout without feeling toy-like
- clear session/activity surfaces
- layered control bars and compact command chips
- dark restrained palette with strong hierarchy
- a feeling of active system supervision rather than generic SaaS panels

## What To Revisit Later

- session/activity panel design
- compact control-band treatment
- operational labeling and state indicators
- information density without losing readability
- whether Olympus needs a stronger “trace / runtime / orchestration” surface in later phases

## What Not To Copy Blindly

- do not import complexity just because it looks advanced
- avoid ornamental control clutter without real function
- keep Olympus centered on projects, Pantheon, and command usefulness rather than mimicry
